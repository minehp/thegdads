Title atau judul dalam bagian header website berguna untuk memberi informasi ke pembaca tentang isi materi yang akan user baca atau akses. Sedangkan metadata berguna untuk memberi informasi ke robot pencari (search engine) agar website yang kita buat bisa index dan dimasukkan ke database search engine seperti google, bing, duckduckgo dan lain-lain. Angular adalah javascript framework yang dapat digunakan untuk membuat static ataupun dinamis website.

Apakah perlu? dan mengapa harus dinamis?

Static website yang digunakan untuk web profil atau blog sangat membutuhkan title dan metadata. Tujuan dari title dan metadata ini agar search engine dengan mudah melakukan indexing sehingga website bisa masuk dalam daftar pencarian user. Meskipun secara teknis implementasi-nya title dan metadata ini tidak bisa secara cepat membuat website langsung di index dan dibaca oleh search engine serta metadata sudah tidak digunakan lagi oleh google untuk web ranking (source) , tetapi saat search engine mendeteksi website yang kita buat, search engine bisa dengan cepat, mudah dan tanpa kendala memasukkan website ke database mereka.

Mengapa harus dinamis...? Alasan utamanya adalah KONTEN, ya konten yang akan di sediakan sebuah website tersebut BUKAN KONTENT STATIS (artinya konten itu dinamis), seperti blog atau web profil yang tiap beberapa periode sekali terjadi perubahan konten. Dinamik title dan metadata harus berganti seiring update page atau update informasi. Konten yang dinamis jika dipasang title dan metadata statis atau sama saja tiap page akan membingungkan user yang ingin mencari informasi.

Bagaimana cara menerapkannya di angular?

Menurut dokumentasi API angular (title dan meta), memasukkan title dan metadata cukup dengan import library yang sudah dimiliki angular ke file component.ts. Nama library-nya @angular/platform-browser, sebenarnya function dari library ini cukup banyak, tapi tidak akan dibahas detil disini, fokus ke Title function dan Meta function saja.

import { Title, Meta } from '@angular/platform-browser';
